## 问题描述

用户在 V2Ray 安装成功后，使用"查看配置"功能（菜单选项 3）时，显示的配置信息与安装时显示的信息不一致。

### 错误对比

| 字段 | 安装时显示（正确） | 查看配置显示（错误） |
|------|-------------------|---------------------|
| 协议 | `vless` | (空) |
| 地址 | `proxy.yourdie.com` | `72.11.140.248` (IP) |
| 用户 ID | 完整 UUID | (空) |
| 传输协议 | `grpc` | `tls` |

---

## 根本原因分析

**文件**: `/home/node/.openclaw/v2ray/src/core.sh`  
**函数**: `get info)` case block (第 1320-1420 行)

### 问题根源

jq 使用逗号操作符 `,` 连接多个表达式时，输出是**换行分隔**（newline-separated），不是逗号分隔：

```bash
# jq 输出示例（每行一个值）：
vless
1080
uuid-1234-5678
grpc
```

但原代码错误地使用 `IFS=',' read -r -a` 尝试按逗号分割：

```bash
# ❌ 错误代码
IFS=',' read -r -a BASE_ARR <<< "$IS_JSON_DATA_BASE"
```

这导致：
1. 所有值被当作一个元素读入数组
2. 后续变量赋值全部错位
3. 最终显示的配置信息完全错误

### 数据流分析

```
1. jq 解析 JSON → 换行分隔输出
   ↓
2. readarray -t 读取 → 正确分割到数组
   ↓
3. 合并所有数组 → ALL_JSON_OUTPUT[]
   ↓
4. 按索引导出变量 → IS_PROTOCOL, PORT, UUID, NET, HOST, etc.
   ↓
5. get addr 设置 IS_ADDR → 使用 HOST 或 fallback 到 IP
   ↓
6. info() 显示 → 根据 IS_INFO_SHOW 索引和 IS_INFO_STR 值
```

## 修复方案

### 代码变更

**文件**: `src/core.sh` (第 1330-1337 行)

```bash
# ❌ 修复前
IFS=',' read -r -a BASE_ARR <<< "$IS_JSON_DATA_BASE"
IFS=',' read -r -a MORE_ARR <<< "$IS_JSON_DATA_MORE"
IFS=',' read -r -a HOST_ARR <<< "$IS_JSON_DATA_HOST"
IFS=',' read -r -a REALITY_ARR <<< "$IS_JSON_DATA_REALITY"

# ✅ 修复后
readarray -t BASE_ARR <<< "$IS_JSON_DATA_BASE"
readarray -t MORE_ARR <<< "$IS_JSON_DATA_MORE"
readarray -t HOST_ARR <<< "$IS_JSON_DATA_HOST"
readarray -t REALITY_ARR <<< "$IS_JSON_DATA_REALITY"
```

### 注释更新

```bash
# ❌ 修复前（误导性注释）
# jq 输出是逗号分隔，需要转换为换行后用 readarray 读取

# ✅ 修复后（正确说明）
# jq 输出是换行分隔，使用 readarray 读取
```

## 经验教训

### Bash + jq 常见陷阱

1. **jq 逗号操作符输出格式**:
   - `(.a),(.b),(.c)` → 换行分隔（每行一个值）
   - `[.a,.b,.c]` → JSON 数组
   - `@csv` → 逗号分隔字符串

2. **正确的解析方式**:
   ```bash
   # 换行分隔输出 → 使用 readarray
   readarray -t arr <<< "$(jq -r '.a,.b,.c')"
   
   # JSON 数组 → 使用 jq -r '.[]'
   readarray -t arr < <(jq -r '.[]')
   
   # 逗号分隔 → 使用 IFS=',' read
   IFS=',' read -r -a arr <<< "$(jq -r '@csv')"
   ```

### 测试建议

- 对于涉及 JSON 解析的代码，必须使用真实 JSON 数据进行测试
- 不要依赖模拟数据，因为格式可能与实际输出不同
- 在 CI/CD 中添加 jq 版本和输出格式的回归测试
